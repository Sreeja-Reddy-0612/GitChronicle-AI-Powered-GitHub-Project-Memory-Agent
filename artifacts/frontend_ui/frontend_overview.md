# Frontend UI Integration (GitChronicle)

## Overview

The frontend layer provides an interactive interface for visualizing repository analysis results generated by the backend.

It transforms raw API responses into structured, user-friendly dashboards.

---

## Tech Stack

- React (Vite)
- Axios (API communication)
- React Router (navigation)
- CSS (custom dark theme)

---

## Architecture

frontend/
│
├── api/ → API communication layer
├── components/ → reusable UI components
├── pages/ → main application views
├── App.jsx → routing
├── main.jsx → entry point

---

## Pages

### Dashboard
- Input GitHub repository URL
- Triggers backend analysis
- Stores response in localStorage

### Insights
Displays repository-level intelligence:
- Most active developer
- Commit type distribution
- Most modified files
- Largest commit

### Phases
- Displays development phases
- Groups commits by evolution stage
- Expandable UI sections

### Commits
- Shows all commits
- Displays classification labels (feature, bugfix, etc.)
- Includes metadata (author, timestamp, SHA)

---

## Data Flow

1. User inputs repo URL
2. Frontend calls backend `/analyze-repo`
3. Backend returns structured JSON
4. Data stored in localStorage
5. Pages read and render data dynamically

---

## UI Features

- Dark theme for readability
- Card-based layout
- Expandable commit groups
- Navigation across insights, phases, commits
- Responsive layout

---

## Impact
